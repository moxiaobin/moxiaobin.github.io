jvm优化

-server 启用JDK的server版本;tomcat默认是以一种叫java –client的模式来运行的, -server即意味着你的tomcat是以真实的production的模式在运行

-Xms 堆的最小内存

-Xmx 堆的最大内存

-Xmn 设置新生代大小

-Xss 栈大小分配

-XX:NewRatio 新生代和老年代比值

-XX:SuvivoRatio 设置两个 Suvivo 和 Eden 区比值

-XX:PermSize 永久代大小( jdk1.8之后移除 )

-XX:MaxPermSize 永久代大小最大值 ( jdk1.8之后移除 )

-XX:MetaspaceSize 元数据区初始值大小( jdk1.8新增 )

-XX:MaxMetaspaceSize 元数据区最大内存( jdk1.8新增 )

-XX:UseSerialGC 串行收集器( 适用单核 CPU)

-XX:UseParNewGC 并行收集器( 适用多核CPU 老年代串行)

-XX:UseParallelGC 并行收集器( 关注吞吐量 老年代串行)

-XX:UseParallelOldGC 并行收集器( 老年代并行)

-XX:MaxGCPauseMills 收集垃圾最大时间,单位毫秒

-XX:UseConsMarkSweepGC CMS 收集器( 单纯老年代收集器 )
